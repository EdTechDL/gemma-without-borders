1. Load the question bank

In plain words

We open our file of 36 quiz questions. Every wrong answer in this file is pre-labelled with the exact mistake a student would have made to pick it. This file was checked by hand and by independent AI verifiers, so we trust it.

2. One door to Gemma

In plain words

This is the only place in the whole project that talks to the AI model. We send it text, it sends text back. Gemma runs on our own computer through Ollama, so no student data ever leaves the machine. To upgrade to the bigger 12B model later, we change one name.

In [2]:
import requests

MODEL = "gemma3:1b"  # later: "gemma3:12b" — one string, nothing else changes

def ask_gemma(prompt):
    r = requests.post("http://localhost:11434/api/generate",
                      json={"model": MODEL, "prompt": prompt,
                            "stream": False,
                            "options": {"temperature": 0.2}})
    return r.json()["response"]

3. Prove the model is alive

In plain words

A quick test question. The answer below was written by Gemma running locally on the laptop, in about four seconds. This is real output, not a mock.

In [3]:
print(ask_gemma("A student calculated 2/3 + 1/4 and got 3/7. "
                "What mistake did they make? Explain for a 14-year-old."))
Out[3]:
Live Gemma output
The mistake was adding the numerators and denominators separately. When we add
fractions, we need the same denominator first. 2/3 becomes 8/12 and 1/4 becomes
3/12, so 8/12 + 3/12 = 11/12. Adding tops and bottoms straight across (2+1 over
3+4) gives 3/7, which is not how fractions work.

4. A student takes the quiz

In plain words

We pretend a student answered five questions and got three wrong. In the real app this comes from the quiz screen — here we just write their choices down so you can see what the agent receives.

In [4]:
quiz = questions[:5]
answers = {
    "NUM-ITEM-01": "B",  # wrong — 2^12 (multiplied the exponents)
    "NUM-ITEM-02": "A",  # correct
    "NUM-ITEM-03": "B",  # wrong — thought 3^0 = 0
    "NUM-ITEM-04": "B",  # wrong — 3/7 (added straight across)
    "NUM-ITEM-05": "A",  # correct
}

5. The agent grades and diagnoses — without the AI

In plain words

Here is our reliability trick. To figure out why the student picked a wrong answer, we do not ask the AI to guess. We look it up in the question file, where every wrong option is already labelled with its mistake. A lookup cannot hallucinate. The AI is saved for the jobs it is actually good at.

In [5]:
from agent import grade_quiz, analyze

result = grade_quiz(quiz, answers)
analysis = analyze(result)

print("Score:", result["correct"], "/", result["total"])
print("Priority trick:", analysis["priority"]["name"])
print("Tell the teacher?", analysis["escalate"])
Out[5]:
Score: 2 / 5
Priority trick: Multiplying exponents when multiplying like bases
Tell the teacher? True
Why this makes it an agent

Notice what just happened: the system looked across all the mistakes, found the pattern, decided which gap matters most, and decided the teacher should be told. It is making decisions about what to do next — that is the difference between an agent and a chatbot that answers one message at a time.

6. Gemma writes the study guide

In plain words

Now the AI does its real job. For each wrong answer, we tell Gemma exactly which mistake was made (from our lookup — so it never teaches the wrong thing) and ask it to write a friendly explanation plus one fresh practice question. This output is from the live model.

In [6]:
from agent import build_study_guides

guides = build_study_guides(result)
print(guides[0]["explanation"][:400])
Out[6]:
Live Gemma output
Okay, let's tackle this together! You were thinking of multiplying the exponents,
which is a common mix-up. When we multiply powers with the same base, we keep the
base and ADD the exponents. So 2^3 x 2^4 = 2^(3+4) = 2^7.

Check it the long way: 2^3 = 8 and 2^4 = 16, and 8 x 16 = 128 — the same as 2^7.

Now you try: simplify 5^2 x 5^3. (Answer: 5^5, which is 3,125.)

7. The teacher hand-off

In plain words

Because this student missed more than half the quiz, the agent writes a short report a teacher can read in ten seconds. This part is plain code, no AI — a safety report should never be a guess.

In [7]:
from agent import teacher_report
print(teacher_report(result, analysis))

That is the whole system. Cells 1, 4, 5, and 7 are plain Python — fast, predictable, impossible to hallucinate. Gemma appears in exactly two places (cells 3 and 6), both through the single ask_gemma() door. Swapping to the 12B model when it finishes downloading is one edit: MODEL = "gemma3:12b". The same cells run unchanged in a Kaggle notebook for the judges — point ask_gemma() at the Kaggle GPU version from our notebook and everything else stays identical.
